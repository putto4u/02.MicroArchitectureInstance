AWS 환경에서 외부 인터넷과의 통신을 관리하는 두 가지 핵심 요소인 **NAT Gateway**와 **EIP(Elastic IP)**에 대해 상세히 정리합니다.

---

## 1. NAT Gateway (Network Address Translation)

NAT 게이트웨이는 프라이빗 서브넷(Private Subnet) 내의 인스턴스가 인터넷에 연결될 수 있도록 돕는 서비스입니다. 보안상 외부에서 내부로 들어오는 직접적인 연결은 차단하면서, 내부에서 외부로 나가는 통신은 허용해야 할 때 사용합니다.

### 주요 특징

* **일방향성**: 내부에서 외부로의 요청은 허용하지만, 인터넷에서 내부 인스턴스로 시작되는 연결은 거부합니다. (보안 강화)
* **관리형 서비스**: AWS가 가용성과 확장성을 자동으로 관리하므로 사용자가 직접 대역폭을 조절할 필요가 없습니다.
* **퍼블릭 서브넷 위치**: NAT 게이트웨이는 반드시 인터넷 게이트웨이(IGW)로의 경로가 있는 **퍼블릭 서브넷**에 생성되어야 합니다.

---

## 2. EIP (Elastic IP, 탄력적 IP)

EIP는 AWS 계정에 할당되는 고정적인 퍼블릭 IPv4 주소입니다. 인스턴스를 중지했다가 다시 시작해도 IP 주소가 바뀌지 않도록 보존할 때 사용합니다.

### 주요 특징

* **고정성**: 동적 IP와 달리 명시적으로 해제하기 전까지는 계정에 고정되어 있습니다.
* **연결성**: EC2 인스턴스, NAT 게이트웨이, Network Load Balancer(NLB) 등에 연결하여 외부 노출용 주소로 사용합니다.
* **이동성**: 특정 인스턴스에 문제가 생겼을 때, EIP를 즉시 다른 예비 인스턴스로 매핑하여 서비스 중단 시간을 최소화할 수 있습니다.

---

## 3. NAT Gateway와 EIP의 관계

NAT 게이트웨이는 생성 시 반드시 **퍼블릭 EIP 1개**를 할당받아야 합니다. 프라이빗 서브넷의 여러 인스턴스가 외부로 나갈 때, 모두 이 NAT 게이트웨이에 할당된 단 하나의 EIP를 대표 주소로 사용하여 인터넷과 통신하게 됩니다.

---

## 4. AWS 과금 및 유료 전환 주의 사항 (2026년 기준)

NAT Gateway와 EIP는 관리 편의성을 제공하지만, 비용 발생에 가장 유의해야 하는 서비스 중 하나입니다.

| **NAT 데이터 처리 요금** | **$0.045 / GB** | 대용량 데이터 전송 시 비용이 급격히 상승함 |
| **퍼블릭 IPv4 (EIP 포함)** | **$0.005 / 시간당 (개당)** | 모든 퍼블릭 IP에 부과 (기존 '미사용 시에만 부과' 정책 폐지) |

> **과금 방지 팁**
> 1. NAT 게이트웨이는 시간당 요금이 비싸므로, 실습 종료 후 반드시 **삭제(Delete)**해야 합니다.
> 2. NAT 게이트웨이를 삭제한 후에는 연결되어 있던 **EIP도 수동으로 '릴리스(Release)'**해야 비용이 멈춥니다. 주소를 할당만 해두고 사용하지 않아도 시간당 비용이 발생합니다.

## 5. 실무 적용 시나리오

1. **DB 업데이트**: 프라이빗 서브넷에 있는 DB 서버가 최신 보안 패치를 받기 위해 외부 레포지토리에 접근해야 할 때 NAT 게이트웨이를 경유합니다.
2. **화이트리스트 등록**: 외부 API 서비스(예: 결제 게이트웨이)를 이용할 때, 우리 서버의 IP를 미리 등록해야 한다면 NAT 게이트웨이에 할당된 고정된 **EIP**를 전달하면 됩니다.
