네트워크 세분화(Segmentation)의 두 핵심 기술인 **서브넷(Subnet)**과 **VLAN**은 모두 "하나의 큰 네트워크를 작게 나누어 관리한다"는 공통적인 목표를 가지고 있습니다. 하지만 이들이 작동하는 계층과 방식에는 명확한 차이가 있습니다.

---

## 1. 서브넷 vs VLAN 비교 요약

| 구분 | 서브넷 (Subnet) | VLAN (Virtual LAN) |
| --- | --- | --- |
| **OSI 계층** | **Layer 3 (Network Layer)** | **Layer 2 (Data Link Layer)** |
| **식별 기준** | **IP 주소** 및 서브넷 마스크 | **VLAN ID** (802.1Q 태그) 및 스위치 포트 |
| **주요 장비** | 라우터 (Router), L3 스위치 | 관리형 스위치 (Switch) |
| **통신 범위** | 서로 다른 서브넷은 라우팅이 필수임 | 동일 VLAN 내에서만 L2 통신 가능 |
| **구동 방식** | 논리적인 IP 대역 분할 | 물리적인 스위치 포트의 가상 분할 |
| **제한 사항** | 클라이언트에서 IP를 수동으로 바꿔 우회 가능 | 스위치 설정에 종속되어 우회가 어려움(보안 우수) |

---

## 2. 공통점: 왜 네트워크를 나누는가?

1. **브로드캐스트 도메인 분리**: 네트워크의 모든 장비에 전달되는 '브로드캐스트' 트래픽의 범위를 제한하여 전체적인 네트워크 성능 저하와 혼잡을 방지합니다.
2. **보안 강화**: 서로 다른 그룹(예: 인사팀 vs 게스트 Wi-Fi)을 격리하여 허가되지 않은 데이터 접근을 차단합니다.
3. **관리 편의성**: 대규모 네트워크를 기능이나 부서별로 작게 쪼개어 장애 발생 시 범위를 한정하고 관리를 용이하게 합니다.

---

## 3. 차이점 상세 해석

### ① 계층적 차이 (Layer 2 vs Layer 3)

* **VLAN(L2)**은 스위치라는 물리적 장비 내부를 '가상의 여러 대의 스위치'가 있는 것처럼 쪼개는 기술입니다. MAC 주소를 기반으로 통신합니다.
* **서브넷(L3)**은 IP 주소라는 논리적 주소 체계를 쪼개는 기술입니다. IP 주소를 기반으로 패킷을 어디로 보낼지 결정합니다.

### ② 구동 방식의 차이

* **VLAN**: 스위치의 특정 포트에 "너는 10번 VLAN이야"라고 이름표(Tag)를 붙입니다. 10번 포트에 꽂힌 컴퓨터는 20번 포트에 꽂힌 컴퓨터와 (같은 스위치에 있어도) 직접 대화할 수 없습니다.
* **서브넷**: 컴퓨터의 운영체제(OS) 설정에서 IP와 서브넷 마스크를 입력하여 결정됩니다.

### ③ 실제 환경에서의 결합 (Best Practice)

실무에서는 보통 **1개의 VLAN에 1개의 서브넷을 매핑**하는 방식을 사용합니다.

* 예: **VLAN 10** (인사팀용 가상 스위치) = **10.1.10.0/24** (인사팀용 IP 대역)
* 이렇게 설정하면 L2(스위치 레벨)와 L3(IP 레벨)에서 모두 완벽하게 격리되며, 라우터(또는 L3 스위치)를 통해서만 서로 통신할 수 있도록 정책을 세울 수 있습니다.

---

## 4. 사례를 통한 이해

* **VLAN만 쓴 경우**: 물리적으로 같은 스위치에 있지만, 스위치 설정으로 대화를 막습니다. 하지만 IP 대역이 같다면 해커가 스위치 설정을 뚫었을 때 바로 모든 정보에 접근 가능합니다.
* **서브넷만 쓴 경우**: IP 대역이 달라도 물리적으로 같은 선(L2)에 연결되어 있다면, 사용자가 자신의 IP 주소를 몰래 바꿔서 다른 팀의 네트워크로 '점프'할 수 있는 보안 취약점이 생깁니다.
* **결론**: 따라서 **VLAN으로 물리적/논리적 통로를 먼저 차단**하고, **서브넷으로 통신 주소 체계를 확립**하는 두 단계를 병행해야 합니다.

---

## 5. AWS 환경에서의 차이 (특이사항)

AWS와 같은 클라우드 환경에서는 사용자가 직접 Layer 2(VLAN)를 만질 수 없습니다.

* 사용자는 **서브넷**을 생성하여 네트워크를 나누고, VLAN의 격리 역할은 **보안 그룹(SG)**이나 **NACL**이 대신 수행하게 됩니다.
