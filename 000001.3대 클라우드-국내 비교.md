## 3대 CSP (AWS, Azure, GCP) 핵심 비교 및 CLI 구축 가이드

본 챕터에서는 글로벌 클라우드 시장을 점유하고 있는 AWS, Microsoft Azure, Google Cloud Platform(GCP)의 인프라 차이점, 서비스 매핑, 그리고 엔지니어링의 기본인 CLI(Command Line Interface) 명령어 구조를 비교 분석합니다.

---

### 1. 주요 서비스 및 용어 매핑 (Service Mapping)

각 벤더는 동일한 기능을 수행하지만 서로 다른 서비스 명칭을 사용합니다. 멀티 클라우드 운영 시 혼동하기 쉬운 핵심 서비스 매핑 테이블입니다.

| 카테고리 | 기능 | **AWS (Amazon Web Services)** | **Microsoft Azure** | **GCP (Google Cloud Platform)** |
| --- | --- | --- | --- | --- |
| **Compute** | 가상 서버 (VM) | **EC2** (Elastic Compute Cloud) | **Azure Virtual Machines** | **Compute Engine** (GCE) |
|  | 컨테이너 관리 (K8s) | **EKS** (Elastic Kubernetes Service) | **AKS** (Azure Kubernetes Service) | **GKE** (Google Kubernetes Engine) |
|  | 서버리스 (FaaS) | **AWS Lambda** | **Azure Functions** | **Cloud Functions** |
| **Storage** | 객체 스토리지 | **S3** (Simple Storage Service) | **Azure Blob Storage** | **Cloud Storage** (GCS) |
|  | 블록 스토리지 (Disk) | **EBS** (Elastic Block Store) | **Azure Disk Storage** | **Persistent Disk** |
|  | 파일 스토리지 (NAS) | **EFS** (Elastic File System) | **Azure Files** | **Filestore** |
| **Network** | 가상 네트워크 | **VPC** (Virtual Private Cloud) | **VNet** (Virtual Network) | **VPC** (Virtual Private Cloud) |
|  | DNS 관리 | **Route 53** | **Azure DNS** | **Cloud DNS** |
|  | 로드 밸런서 | **ELB** (Elastic Load Balancing) | **Azure Load Balancer** | **Cloud Load Balancing** |
| **Database** | RDBMS 관리 | **RDS** (Relational Database Service) | **Azure SQL Database** | **Cloud SQL** |
|  | NoSQL | **DynamoDB** | **Cosmos DB** | **Firestore** / Bigtable |
| **Security** | 계정 권한 관리 | **IAM** | **Microsoft Entra ID** (구 Azure AD) | **Cloud IAM** |

### 2. 벤더별 인프라 철학 및 특화 서비스 분석

#### 2.1 AWS (Amazon Web Services)

* **특징**: 클라우드 시장의 선두주자로서 가장 **방대한 서비스 포트폴리오**와 성숙한 생태계를 보유하고 있습니다. 레퍼런스가 압도적으로 많아 문제 해결이 용이합니다.
* **인프라 구조**: 리전(Region) 중심의 설계가 기본이며, 가용 영역(AZ) 간의 독립성이 매우 철저합니다.
* **특화 서비스**: 서버리스(Lambda), 데이터베이스(DynamoDB, Aurora) 분야에서 기술적 우위를 점하고 있습니다.

#### 2.2 Microsoft Azure

* **특징**: 기존 온프레미스 Windows Server, Active Directory(AD), MSSQL 등을 사용하는 기업에 최적화되어 있습니다. **하이브리드 클라우드** 구축에 가장 강점이 있습니다.
* **인프라 구조**: 리전 수가 가장 많으며, 엔터프라이즈 환경과의 연결성을 중시합니다.
* **특화 서비스**: OpenAI와의 협력을 통한 **Azure OpenAI Service**, GitHub 및 VS Code와의 통합 개발 환경이 강력합니다.

#### 2.3 GCP (Google Cloud Platform)

* **특징**: 구글의 내부 인프라를 사용하므로 데이터 처리 속도와 네트워크 성능이 뛰어납니다. **Kubernetes의 원작자**답게 컨테이너 관리 효율성이 가장 높습니다.
* **인프라 구조**: **글로벌 VPC**가 가장 큰 특징입니다. AWS/Azure는 VPC가 리전(Region)에 종속되지만, GCP는 VPC 하나가 전 세계 리전을 포괄하여 글로벌 라우팅이 훨씬 간결합니다.
* **특화 서비스**: **BigQuery** (초고속 데이터 웨어하우스), AI/ML (Vertex AI, TPU) 분야에서 독보적입니다.

### 3. CLI (Command Line Interface) 구축 및 명령어 비교

클라우드 엔지니어는 웹 콘솔보다 자동화에 유리한 CLI를 주로 사용합니다. 각 벤더의 CLI 문법 스타일(Syntax)은 확연한 차이가 있습니다.

| 구분 | **AWS CLI** | **Azure CLI** | **gcloud CLI (GCP)** |
| --- | --- | --- | --- |
| **설치 도구** | `aws` | `az` | `gcloud` (일부 `gsutil`, `bq`) |
| **문법 구조** | `aws <서비스> <명령어>` | `az <그룹> <하위그룹> <명령어>` | `gcloud <그룹> <하위그룹> <명령어>` |
| **VM 목록 조회** | `$ aws ec2 describe-instances` | `$ az vm list` | `$ gcloud compute instances list` |
| **스토리지 파일 복사** | `$ aws s3 cp local.txt s3://bucket` | `$ az storage blob upload ...` | `$ gsutil cp local.txt gs://bucket` |
| **로그인 명령어** | `$ aws configure` | `$ az login` | `$ gcloud auth login` |
| **출력 포맷 지정** | `--output json`, `--output table` | `--output json`, `--output table` | `--format="json"`, `--format="table"` |

#### 상세 설명 및 실전 팁

* **AWS CLI**: 명령어가 매우 직관적이지는 않습니다 (예: 목록 조회가 `list`가 아닌 `describe-`로 시작하는 경우가 많음). `json` 파싱을 위해 `jq` 툴과 함께 사용하는 것이 필수적입니다.
* **Azure CLI**: 동사-명사 구조가 명확하여 직관적입니다. `az interactive` 모드를 지원하여 자동 완성 및 도움말 기능을 강력하게 지원합니다.
* **gcloud CLI**: 구글 서비스 계층 구조(Project > Service > Resource)를 따릅니다. 스토리지 작업은 `gcloud`가 아닌 `gsutil`, 빅쿼리는 `bq`라는 별도 명령어를 사용하는 등 도구가 나뉘어 있어 주의가 필요합니다.

## 국내 4대 클라우드(CSP) 비교 분석

대한민국의 클라우드 시장은 글로벌 기업(AWS, Azure 등)의 강세 속에서도 국내 기업들이 공공, 금융, 그리고 AI 특화 서비스를 무기로 강력한 입지를 구축하고 있습니다. 국내 4대 CSP(Cloud Service Provider)인 **네이버 클라우드, KT 클라우드, NHN 클라우드, 삼성SDS(또는 카카오엔터프라이즈)**를 중심으로 각 서비스의 특징을 분석합니다.

---

### **1. 국내 4대 클라우드 서비스 비교표**

| 구분 | **네이버 클라우드 (NCP)** | **KT 클라우드 (KT Cloud)** | **NHN 클라우드 (NHN Cloud)** | **삼성SDS (Samsung Cloud Platform)** |
| --- | --- | --- | --- | --- |
| **핵심 강점** | AI(HyperCLOVA X), 국내 최대 서비스 라인업 | 공공/금융 인프라, 국내 최대 IDC 인프라 | 게임/이커머스 특화, 오픈스택 기술력 | 기업용(Enterprise) 보안, 하이브리드 클라우드 |
| **주요 타겟** | 중소기업(SMB), 스타트업, 교육 | 공공기관, 금융권, 대형 엔터프라이즈 | 게임사, 이커머스, 공공(G-Cloud) | 삼성 계열사, 글로벌 제조/물류 대기업 |
| **AI 전략** | 생성형 AI 기반의 비즈니스 도구 제공 | MS 협력을 통한 시큐어 퍼블릭 클라우드 | 엔비디아 기반 GPU 클라우드 강화 | 기업 맞춤형 AI 서비스 및 클라우드 보안 |
| **인프라 규모** | 글로벌 리전 보유 (일본, 독일 등) | 전국 13개 이상 IDC 운영 (국내 최대) | 판교/광주 데이터센터 (GPU 중심) | 전 세계 삼성 데이터센터 거점 활용 |

---

### **2. 서비스별 상세 분석**

**01. 네이버 클라우드 (Naver Cloud Platform)**

* **특징:** 네이버의 검색, 메일, 메신저를 운영해온 노하우를 클라우드에 이식했습니다. 국내 기업 중 가장 많은 200개 이상의 상품을 제공합니다.
* **강점:** 한국어 특화 생성형 AI인 **HyperCLOVA X**를 가장 쉽게 연동할 수 있어 AI 서비스 도입을 원하는 기업에 유리합니다.

**02. KT 클라우드 (KT Cloud)**

* **특징:** 국내 최초로 보안 인증(CSAP)을 획득하며 공공 클라우드 시장을 개척했습니다. 통신 인프라와 결합된 하이브리드 클라우드에 강점이 있습니다.
* **강점:** 전국에 퍼져 있는 물리적인 데이터센터(IDC)와 전용선을 통해 네트워크 안정성과 대규모 트래픽 처리에 탁월합니다.

**03. NHN 클라우드 (NHN Cloud)**

* **특징:** 게임 포털 '한게임'과 결합된 안정적인 인프라 기술력을 바탕으로 성장했습니다. 유연한 오픈스택(OpenStack) 기반으로 설계되었습니다.
* **강점:** 국가 AI 데이터센터(광주)를 운영하며 고성능 컴퓨팅(HPC) 및 GPU 인프라를 필요로 하는 AI 연산 최적화에 집중하고 있습니다.

**04. 삼성SDS (Samsung Cloud Platform)**

* **특징:** 삼성 그룹의 IT 서비스를 책임지며 쌓은 강력한 **엔터프라이즈 보안**과 신뢰성이 핵심입니다.
* **강점:** 복잡한 기업 환경(On-premise + Cloud)을 연결하는 하이브리드 클라우드 설계 및 글로벌 물류/제조 시스템 운영 능력에 특화되어 있습니다.

---

### **3. 국내 CSP 선택 시 고려사항 [비용 및 보안]**

1. **공공/금융 보안 준수:** 국내 기업들은 **CSAP(클라우드 서비스 보안인증)**를 보유하고 있어, 공공기관이나 엄격한 규제가 적용되는 금융권 프로젝트에 필수적입니다.
2. **기술 지원 및 언어:** 글로벌 CSP와 달리 한국어 기술 지원이 매우 빠르고 상세하며, 국내 비즈니스 환경(결제 체계, 세금 계산 등)에 최적화되어 있습니다.
3. **[유료 과금 주의]** * 국내 CSP들도 글로벌 기업과 유사하게 **데이터 전송료(Outbound Traffic)**와 **스토리지 사용량**에 따라 비용이 증가합니다.
* 특정 기업용 특화 솔루션(예: 그룹웨어 연동, AI 모델 학습)은 도입 즉시 높은 고정비가 발생할 수 있으므로 사전 견적이 필수입니다.
