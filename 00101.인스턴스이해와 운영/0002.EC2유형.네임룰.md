# Amazon EC2 인스턴스 유형 (Instance Types) 및 명명 규칙

## 1. 인스턴스 명명 규칙 (Naming Convention)

AWS 인스턴스 이름은 단순한 모델명이 아니라, 해당 서버의 스펙과 세대를 나타내는 암호와 같다. 이를 해독할 수 있어야 적합한 서버를 선택할 수 있다.

**형식:** `[패밀리][세대][추가기능].[크기]`
**예시:** **`m5d.large`**

1. **패밀리 (Family):** 인스턴스의 주 용도를 나타내는 알파벳. (예: `m` = 범용, `c` = 컴퓨팅, `r` = 메모리)
2. **세대 (Generation):** 숫자가 높을수록 최신 기술이 적용된 모델. (예: `5`가 `4`보다 최신, 성능 및 가성비 우수)
3. **추가 기능 (Attribute):** (선택 사항) 특수 하드웨어 속성.
* `d`: NVMe SSD(로컬 디스크) 포함
* `g`: AWS Graviton 프로세서(ARM 기반) 사용
* `a`: AMD 프로세서 사용 (Intel 대비 저렴)


4. **크기 (Size):** 서버의 물리적 자원 규모. (예: `large`, `xlarge`, `2xlarge` 등). 크기가 한 단계 커질 때마다 vCPU와 메모리는 2배씩 증가한다.

## 2. 인스턴스 패밀리 분류 (Instance Families)

AWS는 워크로드(작업 부하)의 특성에 따라 인스턴스를 크게 5가지 카테고리로 분류한다. 각 패밀리의 앞글자를 따서 외우는 것이 일반적이다.

### 2.1 범용 (General Purpose) - 밸런스형

CPU, 메모리, 네트워크 리소스가 균형 있게 구성된 모델.

* **대표 패밀리:** **M** (Main), **T** (Turbo/Burstable)
* **특징:**
* **M 시리즈:** 웹 서버, 애플리케이션 서버, 중소형 데이터베이스 등 가장 무난하게 사용됨.
* **T 시리즈:** 평소 사용량이 낮고 가끔 튀는 워크로드에 적합 (개발 서버, 코드 저장소 등). **[💰T3 Unlimited 과금 주의]**


* **권장 대상:** 트래픽을 예측하기 어렵거나 다양한 작업을 처리해야 하는 경우.

### 2.2 컴퓨팅 최적화 (Compute Optimized) - CPU 강화형

메모리 대비 고성능 vCPU 비율이 높은 모델.

* **대표 패밀리:** **C** (Compute)
* **특징:** 연산 능력이 가장 중요할 때 사용. 동일 비용 대비 CPU 성능이 가장 우수함.
* **권장 대상:**
* 고성능 웹 서버 (HPC)
* 배치(Batch) 처리 작업
* 동영상 인코딩, 게임 서버
* 머신러닝 추론 (Inference)



### 2.3 메모리 최적화 (Memory Optimized) - RAM 강화형

CPU 대비 대용량 메모리를 탑재한 모델.

* **대표 패밀리:** **R** (RAM), **X** (Xtreme Memory)
* **특징:** 데이터를 메모리에 올려두고 빠르게 처리해야 하는 작업에 특화됨.
* **권장 대상:**
* 고성능 관계형 데이터베이스 (RDS의 주력 모델)
* NoSQL 데이터베이스 (MongoDB, Cassandra 등)
* In-Memory 캐시 (Redis, Memcached)
* 실시간 빅데이터 분석 (Apache Spark)



### 2.4 스토리지 최적화 (Storage Optimized) - 디스크 I/O 강화형

로컬 스토리지의 초당 입출력(IOPS)이 매우 높고 용량이 큰 모델.

* **대표 패밀리:** **I** (I/O), **D** (Dense)
* **특징:** EBS가 아닌 호스트 서버에 직접 연결된 고성능 SSD/HDD를 제공하여 I/O 병목을 제거함.
* **권장 대상:**
* 초고속 I/O가 필요한 NoSQL (I 시리즈)
* 대용량 데이터 웨어하우징 (D 시리즈)
* 분산 파일 시스템



### 2.5 가속 컴퓨팅 (Accelerated Computing) - GPU/FPGA 탑재 **[💰 고비용 주의]**

CPU만으로는 처리가 불가능한 작업을 위해 하드웨어 가속기(GPU, FPGA)를 탑재한 모델.

* **대표 패밀리:** **P** (Performance/GPU), **G** (Graphics), **Inf** (Inference)
* **특징:** NVIDIA GPU 등이 장착되어 있으며 시간당 비용이 매우 비쌈.
* **권장 대상:**
* **P 시리즈:** 딥러닝/머신러닝 모델 학습 (Training)
* **G 시리즈:** 그래픽 렌더링, 3D 시각화, 비디오 스트리밍
* **Inf 시리즈:** 머신러닝 추론 전용 (가성비 중심)

## 3. 요약 및 선택 가이드 (Cheat Sheet)

| 유형 | 패밀리 코드 | 주요 특징 | 추천 워크로드 |
| --- | --- | --- | --- |
| **범용** | **M, T** | CPU:RAM = 1:4 (균형) | 웹 서버, 개발 환경, 소규모 DB |
| **컴퓨팅** | **C** | CPU:RAM = 1:2 (CPU 높음) | 배치 처리, 인코딩, 고성능 웹 |
| **메모리** | **R, X** | CPU:RAM = 1:8 이상 (RAM 높음) | **데이터베이스**, 캐시 서버 |
| **스토리지** | **I, D** | 높은 Disk I/O, 대용량 로컬 디스크 | NoSQL, 데이터 웨어하우스 |
| **가속** | **P, G** | **[💰고가]** GPU 탑재 | AI/ML 학습, 3D 렌더링 |
