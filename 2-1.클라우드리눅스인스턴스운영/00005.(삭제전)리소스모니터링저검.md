## 관리자 관점의 리소스 모니터링 및 상태 점검 (삭제 전 필수 확인)

EC2 인스턴스를 종료(삭제)하기 전, 관리자는 해당 리소스의 현재 상태, 성능 지표, 연결된 종속 리소스, 그리고 발생 비용을 정확히 파악해야 합니다. 이 장에서는 무작정 삭제 버튼을 누르기 전, 콘솔에서 확인해야 할 **4가지 핵심 관리 포인트**를 다룹니다.

---

### 1. 인스턴스 상태 검사 (Status Checks) 및 세부 정보

EC2 대시보드에서 인스턴스를 선택하면 하단 패널을 통해 서버의 건강 상태를 정밀하게 진단할 수 있습니다.

#### 1.1 상태 검사 (Status Checks) 탭

AWS는 1분마다 서버의 상태를 자동으로 모니터링합니다. 관리자는 **'2/2개 검사 통과'** 여부를 반드시 확인해야 합니다.

* **시스템 상태 검사 (System Status Checks):** AWS의 물리적 하드웨어, 전원, 네트워크 문제 여부입니다. 여기서 실패가 뜨면 AWS 측의 장애이므로 사용자가 해결할 수 없습니다. (Stop & Start로 하드웨어 이동 필요)
* **인스턴스 상태 검사 (Instance Status Checks):** OS 내부의 문제입니다. 부팅 실패, 메모리 부족, 커널 패닉 등이 원인이며, 사용자가 직접 SSH로 접속하거나 시스템 로그를 확인하여 해결해야 합니다.

#### 1.2 세부 정보 (Details) 탭

서버의 정체성을 확인합니다.

* **퍼블릭/프라이빗 IP:** 현재 할당된 IP 주소 확인.
* **AMI ID:** 이 서버가 어떤 운영체제 이미지를 기반으로 생성되었는지 확인. (추후 동일한 서버 복제 시 필요)
* **IAM 역할:** 서버에 부여된 권한 확인. (중요한 권한을 가진 서버를 삭제하면 연동된 서비스가 중단될 수 있음)

### 2. 모니터링 탭을 통한 성능 지표 (CloudWatch Metrics)

**[모니터링(Monitoring)]** 탭을 클릭하면 서버의 CPU, 네트워크, 디스크 사용량을 그래프로 볼 수 있습니다. 삭제 전 이 서버가 **'유휴(Zombie) 자원'**인지 **'실사용 자원'**인지 판단하는 근거가 됩니다.

* **CPU 사용률 (CPU Utilization):** 그래프가 지속적으로 1~5% 미만이라면 거의 사용되지 않는 서버일 가능성이 높습니다.
* **네트워크 입/출력 (Network In/Out):** 트래픽이 꾸준히 발생하고 있다면 외부에서 누군가 접속 중이거나 백그라운드 작업이 도는 중이므로 삭제 시 주의해야 합니다.

> **※ 과금 안내 (CloudWatch):** 기본 모니터링(5분 주기)은 **무료**입니다. '세부 모니터링 활성화(1분 주기)'를 켜면 **추가 요금**이 발생하므로 필요한 경우에만 활성화합니다.

---

### 3. 스토리지 및 연결 리소스 확인 (고아 리소스 방지)

EC2 인스턴스를 삭제해도, 설정에 따라 **EBS 볼륨(하드디스크)**이나 **탄력적 IP(Elastic IP)**는 삭제되지 않고 남아 계속 요금을 발생시킬 수 있습니다. 이를 **'고아 리소스(Orphaned Resources)'**라고 합니다.

#### 3.1 스토리지 (Storage) 탭

* **종료 시 삭제 (Delete on Termination):** 블록 디바이스 정보에서 이 항목이 **'예(Yes)'**로 되어 있는지 확인합니다.
* **Yes:** 인스턴스 삭제 시 디스크도 같이 삭제됩니다. (데이터 유실 주의)
* **No:** 인스턴스만 삭제되고 디스크(EBS)는 남아 **계속 과금**됩니다. 데이터 보존이 목적이 아니라면 삭제 시 수동으로 EBS도 지워야 합니다.



#### 3.2 탄력적 IP (Elastic IP)

인스턴스 요약 정보에 탄력적 IP가 연결되어 있다면, 인스턴스 삭제 후 반드시 **[네트워크 및 보안] > [탄력적 IP]** 메뉴로 이동하여 IP를 **[릴리스(Release/반납)]**해야 합니다. AWS는 사용하지 않고 점유만 하고 있는 탄력적 IP에 대해 **시간당 페널티 요금**을 부과합니다.

### 4. 청구 대시보드 및 태그 관리

#### 4.1 결제 대시보드 (Billing Dashboard)

우측 상단 계정 메뉴에서 **[결제 대시보드]**로 이동합니다.

* **Free Tier 사용량:** 프리 티어 한도(EC2 750시간/월, EBS 30GB 등)를 얼마나 소진했는지 그래프로 보여줍니다.
* **예상 청구액:** 현재까지 발생한 요금과 월말 예상 요금을 확인합니다. (※ 데이터 반영에 약 24시간의 지연이 있을 수 있습니다.)

#### 4.2 태그 (Tags) 탭

이 서버가 어떤 프로젝트, 어떤 환경(Dev/Prod), 누가 담당자인지 명시된 '꼬리표'를 확인합니다.

* 협업 환경에서는 태그를 확인하지 않고 삭제할 경우, 타 부서의 중요 테스트 서버를 날리는 사고가 발생할 수 있습니다. `Environment: Production` 태그가 붙은 리소스는 절대 임의로 삭제해서는 안 됩니다.

### 요약: 삭제 전 관리자 체크리스트

1. **상태 검사:** `2/2 통과` 상태인가? (정상 작동 중인 서버를 끄는 것인지 확인)
2. **모니터링:** CPU/네트워크 사용량이 `0`에 수렴하는가? (사용 중인 트래픽이 없는지 확인)
3. **종속성:** `종료 시 삭제(Delete on Termination)` 옵션이 켜져 있는가? (EBS 잔존 과금 방지)
4. **IP:** 탄력적 IP(EIP)가 연결되어 있는가? (삭제 후 IP 반납 필수)
