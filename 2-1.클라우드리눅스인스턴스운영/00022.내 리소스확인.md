EC2 인스턴스를 생성한 후, 내 리소스가 어디에 있는지 확인하는 방법은 크게 **콘솔(Web)**과 **CLI(명령어)** 두 가지가 있습니다.

가장 흔한 실수는 **'리전(Region)'이 다르게 설정되어 있어 생성된 서버가 보이지 않는 경우**입니다.

---

### 1. AWS 웹 콘솔에서 확인하기 (가장 기본)

GUI 화면에서 눈으로 직접 확인하는 방법입니다.

1. **EC2 대시보드 진입:** 콘솔 상단 검색창에 `EC2`를 입력하고 선택합니다.
2. **인스턴스 메뉴 클릭:** 왼쪽 사이드바 메뉴에서 **[인스턴스(Instances)]**를 클릭합니다.
3. **목록 확인:** 중앙 화면에 내가 만든 서버의 이름, 인스턴스 ID, 상태(Running/Stopped)가 표출됩니다.

#### ⚠️ 주의: 리소스가 안 보일 때 (가장 중요)

분명히 생성했는데 목록이 **'없음(No instances found)'**으로 나온다면 99% **리전(Region)** 문제입니다.

* **확인 위치:** 콘솔 화면 **우측 상단**에 지역 이름(예: `US East (N. Virginia)`)이 무엇으로 되어 있는지 확인하세요.
* **해결:** 내가 서버를 생성했던 리전(예: `아시아 태평양(서울) ap-northeast-2`)으로 변경해야만 리소스가 보입니다.

---

### 2. AWS CLI로 확인하기 (빠른 조회)

터미널에서 명령어를 통해 현재 리전의 인스턴스 목록을 조회합니다.

**기본 조회 명령어:**

```bash
aws ec2 describe-instances

```

**보기 좋게 정리해서 조회 (테이블 형식):**
JSON 형식이 너무 길어 보기 힘들다면, 아래 명령어를 사용하여 **이름, 인스턴스ID, 상태, 공인IP**만 깔끔하게 출력할 수 있습니다.

```bash
aws ec2 describe-instances --query "Reservations[*].Instances[*].{Name:Tags[?Key=='Name']|[0].Value,Instance:InstanceId,Status:State.Name,IP:PublicIpAddress}" --output table

```

**특정 리전 강제 조회:**
내 CLI 설정(default)과 다른 리전을 보고 싶다면 `--region` 옵션을 붙입니다.

```bash
aws ec2 describe-instances --region ap-northeast-2

```

---

### 3. "어디에 만들었는지 모르겠어요" (전체 리전 검색)

만약 어느 리전에 만들었는지 기억나지 않는다면 **'리소스 그룹 및 태그 편집기'**를 사용해 전 세계 리전을 한 번에 뒤질 수 있습니다.

1. 콘솔 검색창에 **`Resource Groups & Tag Editor`** (리소스 그룹 및 태그 편집기) 입력 후 이동.
2. 좌측 메뉴 **[태그 편집기(Tag Editor)]** 선택.
3. **리전(Regions):** '모든 리전(All regions)' 선택.
4. **리소스 유형(Resource types):** `AWS::EC2::Instance` 선택.
5. **[리소스 검색(Search resources)]** 버튼 클릭.
* 결과 화면에 전 세계에 흩어진 내 EC2 인스턴스가 모두 표시됩니다.
