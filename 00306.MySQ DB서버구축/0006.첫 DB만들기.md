# 첫 데이터베이스 만들기

강의 교재의 첫 번째 실습으로, 실제로 데이터를 저장할 수 있는 공간인 데이터베이스를 설계하고 생성하는 과정을 학습합니다.

---

## 1. 데이터베이스 설계 (Modeling)

데이터베이스를 만들기 전, 어떤 데이터를 담을지 설계하는 과정이 필요합니다. 이를 **데이터 모델링**이라고 합니다. 가장 보편적인 예시인 '회원 관리 시스템'을 기준으로 설계해 보겠습니다.

### 요구사항 분석

* 회원의 고유 번호가 필요하다.
* 회원의 이름, 이메일, 가입일을 저장해야 한다.
* 이메일은 중복될 수 없다.

## 2. 데이터베이스 및 테이블 생성 (DDL)

설계가 끝났다면 SQL 명령어를 통해 실제 저장 공간을 생성합니다.

### 2.1 데이터베이스 생성

여러 개의 테이블을 담는 큰 주머니인 데이터베이스(Schema)를 생성합니다.

```sql
-- 'my_shop'이라는 이름의 데이터베이스 생성
CREATE DATABASE my_shop;

-- 생성한 데이터베이스 사용 선언
USE my_shop;

```

### 2.2 테이블 생성

실제 데이터가 저장될 표(Table)를 생성합니다. 각 컬럼에는 저장될 데이터의 종류(데이터 타입)를 지정해야 합니다.

```sql
CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,   -- 자동으로 증가하는 고유 번호 (기본키)
    name VARCHAR(50) NOT NULL,           -- 최대 50자의 문자열 (빈 값 허용 안 함)
    email VARCHAR(100) UNIQUE NOT NULL,  -- 최대 100자의 문자열 (중복 불가)
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP -- 가입 일자 (현재 시간 자동 입력)
);

```

---

## 3. 데이터 입력 및 확인 (DML)

만들어진 테이블에 첫 번째 데이터를 채워 넣습니다.

```sql
-- 데이터 삽입
INSERT INTO users (name, email) 
VALUES ('홍길동', 'hong@example.com');

-- 데이터 확인
SELECT * FROM users;

```

## 4. 클라우드에서 시작하기 (AWS RDS)

내 컴퓨터가 아닌 실제 서버 환경에서 데이터베이스를 구축하려면 **AWS RDS**를 사용합니다.

### AWS RDS 생성 시 주의사항

* **엔진 선택**: MySQL 또는 PostgreSQL을 주로 선택합니다.
* **프리티어 확인**: 실습 시 반드시 '프리티어(Free Tier)' 옵션을 체크해야 비용 발생을 막을 수 있습니다.
* **과금 주의**:
* 인스턴스를 생성하면 사용 여부와 상관없이 **시간당 비용**이 산정됩니다. (프리티어는 월 750시간 무료)
* **할당된 스토리지(용량)**가 프리티어 범위를 초과하면 **추가 비용이 즉시 발생**합니다.
* 실습이 끝나고 인스턴스를 '중지'해도 데이터 저장 비용은 계속 발생하므로, 더 이상 쓰지 않는다면 **'삭제(Terminate)'**해야 합니다.
