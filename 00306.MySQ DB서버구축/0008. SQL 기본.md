# Chapter 6-1. SQL(Structured Query Language)의 개요와 핵심 문법

SQL(Structured Query Language)은 관계형 데이터베이스 관리 시스템(RDBMS)에서 데이터를 정의, 조작, 제어하기 위해 사용하는 **표준 프로그래밍 언어**입니다.

사용자는 데이터가 물리적으로 어디에 저장되어 있는지 알 필요 없이, SQL을 통해 "무엇(What)을 원하는지"만 기술하면 DBMS가 이를 처리합니다.

---

### 1. SQL 명령어의 4가지 분류

SQL 문장은 그 기능과 목적에 따라 크게 4가지로 분류됩니다. 데이터베이스 학습 시 가장 먼저 이해해야 할 구조입니다.

| 분류 | 약어 (Full Name) | 설명 | 주요 명령어 |
| --- | --- | --- | --- |
| **데이터 조작어** | **DML** (Data Manipulation Language) | 데이터의 검색, 삽입, 수정, 삭제를 수행합니다. 개발자가 가장 빈번하게 사용합니다. | `SELECT`, `INSERT`, `UPDATE`, `DELETE` |
| **데이터 정의어** | **DDL** (Data Definition Language) | 테이블, 인덱스 등 DB의 구조(Schema)를 생성하거나 변경합니다. | `CREATE`, `ALTER`, `DROP`, `TRUNCATE` |
| **데이터 제어어** | **DCL** (Data Control Language) | 데이터 접근 권한을 부여하거나 박탈합니다. 보안과 관련이 깊습니다. | `GRANT`, `REVOKE` |
| **트랜잭션 제어어** | **TCL** (Transaction Control Language) | DML에 의한 데이터 변경 사항을 확정하거나 취소하여 데이터 무결성을 유지합니다. | `COMMIT`, `ROLLBACK`, `SAVEPOINT` |

### 2. CRUD와 기본 문법 실습

애플리케이션 개발의 핵심 기능인 **CRUD(Create, Read, Update, Delete)**는 SQL의 DML과 1:1로 대응됩니다.

> **예제 시나리오:** 직원 정보를 저장하는 `employees` 테이블을 기준으로 합니다.

#### 2.1 데이터 조회 (Read) - `SELECT`

데이터베이스에서 가장 많이 사용되는 명령어입니다.

* **기본 구조:** `SELECT [컬럼] FROM [테이블] WHERE [조건];`

```sql
-- employees 테이블의 모든 컬럼(*) 조회
SELECT * FROM employees;

-- 특정 컬럼(이름, 연봉)만 조회하며, 부서가 'IT'인 직원만 필터링
SELECT name, salary
FROM employees
WHERE department = 'IT';

```

#### 2.2 데이터 삽입 (Create) - `INSERT`

테이블에 새로운 행(Row)을 추가합니다.

* **기본 구조:** `INSERT INTO [테이블](컬럼1, 컬럼2) VALUES (값1, 값2);`

```sql
-- 새로운 직원 '홍길동'을 IT 부서에 추가
INSERT INTO employees (id, name, department, salary)
VALUES (101, 'Hong Gil-Dong', 'IT', 5000);

```

#### 2.3 데이터 수정 (Update) - `UPDATE`

이미 저장된 데이터를 변경합니다.

* **주의:** `WHERE` 절을 생략하면 테이블 내 **모든 데이터**가 수정되는 대형 사고가 발생할 수 있습니다.
* **기본 구조:** `UPDATE [테이블] SET [컬럼]=[변경값] WHERE [조건];`

```sql
-- ID가 101인 직원의 연봉을 5500으로 인상
UPDATE employees
SET salary = 5500
WHERE id = 101;

```

#### 2.4 데이터 삭제 (Delete) - `DELETE`

테이블의 행을 삭제합니다.

* **주의:** `UPDATE`와 마찬가지로 `WHERE` 절 없이 실행하면 모든 데이터가 삭제됩니다.
* **기본 구조:** `DELETE FROM [테이블] WHERE [조건];`

```sql
-- ID가 101인 직원의 정보 삭제
DELETE FROM employees
WHERE id = 101;

```

### 3. SQL 작성 규칙 및 특징

SQL을 오류 없이 작성하기 위해 지켜야 할 기본적인 규칙들입니다.

1. **세미콜론(`;`)**
* SQL 문장의 끝은 반드시 세미콜론으로 맺어야 합니다. (대부분의 SQL 클라이언트 툴에서 필수)


2. **대소문자 구분 없음 (Case Insensitive)**
* SQL 키워드(`SELECT`, `FROM` 등)는 대소문자를 구분하지 않습니다.
* **권장 컨벤션:** 가독성을 위해 **명령어는 대문자**, 테이블 및 컬럼명은 **소문자**로 작성하는 것이 업계 표준입니다.
* *주의:* 데이터의 '값' (예: 'Hong Gil-Dong')은 대소문자를 엄격히 구분합니다.


3. **주석 (Comment)**
* 한 줄 주석: `--` (하이픈 두 개)
* 여러 줄 주석: `/* 내용 */`
