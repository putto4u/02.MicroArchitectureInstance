# 데이터베이스(Database) 입문

## 1. 데이터베이스의 정의와 필요성

**데이터베이스(Database)**는 여러 사람이 공유하여 사용할 목적으로 체계적으로 구조화하여 저장한 데이터의 집합입니다. 단순히 데이터를 모아두는 것을 넘어, 효율적인 관리와 검색이 가능하도록 설계된 시스템을 의미합니다.

### 왜 데이터베이스를 사용해야 할까요?

* **데이터 중복 최소화**: 동일한 데이터가 여러 곳에 저장되는 것을 방지합니다.
* **데이터 무결성 유지**: 데이터의 정확성과 일관성을 보장합니다.
* **보안성**: 권한이 있는 사용자만 데이터에 접근할 수 있도록 제어합니다.
* **동시 제어**: 여러 사용자가 동시에 같은 데이터에 접근해도 충돌이 발생하지 않도록 관리합니다.

---

## 2. 데이터베이스 관리 시스템 (DBMS)

데이터베이스를 직접 관리하고 사용자의 요구에 따라 정보를 생성해 주는 소프트웨어를 **DBMS(Database Management System)**라고 합니다.

### 대표적인 DBMS의 종류

* **관계형 (RDBMS)**: 데이터를 표(Table) 형태로 관리합니다. (예: MySQL, Oracle, PostgreSQL, SQL Server)
* **비관계형 (NoSQL)**: 정해진 형식이 없는 대용량 데이터를 유연하게 관리합니다. (예: MongoDB, Redis, Cassandra)

---

## 3. 관계형 데이터베이스의 구조

관계형 데이터베이스는 데이터를 행(Row)과 열(Column)로 이루어진 2차원 표 형태로 저장합니다.

* **Table (테이블)**: 데이터를 담는 표 전체를 의미합니다.
* **Column (컬럼/열)**: 데이터의 속성(Attribute)입니다. (예: 이름, 나이, 전화번호)
* **Row (로우/행)**: 개별적인 데이터 한 건(Record/Tuple)입니다.
* **Primary Key (기본키)**: 각 행을 유일하게 식별할 수 있는 핵심 값입니다. (예: 회원번호, 주민등록번호)

---

## 4. SQL (Structured Query Language)

SQL은 관계형 데이터베이스와 통신하기 위해 사용하는 표준 언어입니다. 크게 세 가지 카테고리로 나뉩니다.

### 4.1 DDL (Data Definition Language)

데이터베이스의 구조(스키마)를 정의하거나 변경합니다.

* `CREATE`: 테이블 생성
* `ALTER`: 테이블 구조 수정
* `DROP`: 테이블 삭제

### 4.2 DML (Data Manipulation Language)

데이터를 직접 조작(삽입, 조회, 수정, 삭제)합니다.

* `SELECT`: 데이터 조회
* `INSERT`: 데이터 삽입
* `UPDATE`: 데이터 수정
* `DELETE`: 데이터 삭제

### 4.3 DCL (Data Control Language)

데이터에 대한 접근 권한을 관리합니다.

* `GRANT`: 권한 부여
* `REVOKE`: 권한 취소

## 5. 클라우드 데이터베이스 (AWS)

현대 인프라에서는 물리적인 서버 대신 클라우드 데이터베이스를 주로 사용합니다.

### AWS RDS (Relational Database Service)

AWS에서 제공하는 관리형 RDBMS 서비스입니다. 설치, 패치, 백업 등을 AWS가 자동화해 줍니다.

> **과금 주의**:
> * 무료 계정(프리티어) 범위를 벗어나는 고성능 인스턴스 사용 시 **즉시 과금**됩니다.
> * 사용하지 않는 RDS 인스턴스를 켜둘 경우 시간에 따라 **비용이 지속적으로 발생**하므로 실습 후 반드시 삭제하거나 중지해야 합니다.
> 
> 

### AWS DynamoDB

완전 관리형 NoSQL 데이터베이스 서비스입니다.

> **과금 주의**:
> * 읽기/쓰기 용량 설정에 따라 **비용이 발생**하며, 데이터 저장 용량이 늘어날수록 **유료 전환 및 추가 비용**이 발생할 수 있습니다.
> 
>
