MariaDB와 MySQL은 현대 웹 생태계에서 가장 널리 쓰이는 오픈 소스 관계형 데이터베이스 관리 시스템(RDBMS)입니다. 두 데이터베이스의 관계와 설치 방법은 다음과 같습니다.

---

## MariaDB와 MySQL의 관계

MariaDB는 **MySQL의 포크(Fork)** 버전입니다. 즉, MySQL의 초기 개발자인 마이클 "몬티" 위데니우스가 MySQL이 오라클(Oracle)에 인수된 후, 오픈 소스 정신을 유지하기 위해 기존 소스 코드를 기반으로 새롭게 만든 데이터베이스입니다.

### 1. 호환성 (Drop-in Replacement)

MariaDB는 초기 설계부터 MySQL의 '대체제'를 목표로 개발되었습니다.

* **명령어 동일**: `mysql`, `mysqldump` 등의 명령어를 그대로 사용합니다.
* **포트 번호**: 기본적으로 **3306번** 포트를 공유합니다.
* **커넥터 호환**: Python, Java, PHP 등에서 사용하던 기존 MySQL 라이브러리를 그대로 사용하여 MariaDB에 접속할 수 있습니다.

### 2. 주요 차이점

| 구분 | MySQL | MariaDB |
| --- | --- | --- |
| **관리 주체** | Oracle (영리 기업) | MariaDB Foundation (비영리 재단) |
| **라이선스** | GPL / 상용 라이선스 (이원화) | **완전 오픈 소스 (GPL)** |
| **성능** | 읽기 중심 작업에 최적화 | 복잡한 쿼리 및 대규모 연결 처리에서 우위 |
| **특징** | 오라클 기술과의 통합성 강조 | 다양한 스토리지 엔진(Aria, MyRocks 등) 제공 |

> [!IMPORTANT]
> **과금 주의 사항**
> * **MySQL**: 커뮤니티 버전은 무료이나, 엔터프라이즈 기능(스레드 풀링, 고급 보안 등)을 사용하려면 **유료 라이선스**를 구매해야 합니다.
> * **MariaDB**: 모든 기능이 오픈 소스로 제공되지만, 기업용 기술 지원이 포함된 **MariaDB Enterprise**는 유료 서비스입니다.
> * **클라우드(AWS/Azure)**: AWS RDS 등에서 해당 DB를 사용할 경우 인스턴스 사용료 및 데이터 전송료가 발생합니다.

## 다운로드 및 설치 방법

### 1. MariaDB 설치 (Windows 기준)

1. **공식 사이트 접속**: [MariaDB 다운로드 페이지](https://mariadb.org/download/)로 이동합니다.
2. **버전 선택**: 최신 안정화 버전(Stable, 예: 11.4 이상)을 선택하고 `MSI` 패키지를 다운로드합니다.
3. **설치 실행**:
* `Next`를 눌러 진행하다가 **Root password** 설정 단계에서 관리자 비밀번호를 입력합니다.
* `Install as service` 체크박스를 확인합니다 (부팅 시 자동 실행).
* 포트 번호는 기본값인 **3306**을 사용합니다.


4. **완료**: 설치 후 `HeidiSQL`(동봉됨) 또는 터미널에서 접속을 확인합니다.

### 2. MySQL 설치 (Windows 기준)

1. **공식 사이트 접속**: [MySQL Community Downloads](https://dev.mysql.com/downloads/installer/)로 이동합니다.
2. **Installer 다운로드**: `MySQL Installer for Windows`를 다운로드하여 실행합니다.
3. **Setup Type**: `Developer Default` 또는 서버만 설치하려면 `Server only`를 선택합니다.
4. **Check Requirements**: 필요한 Visual C++ 라이브러리가 없다면 `Execute`를 눌러 먼저 설치합니다.
5. **Configuration**:
* **Type and Networking**: 'Development Computer'로 설정.
* **Authentication**: 'Use Strong Password Encryption' 권장.
* **Accounts**: Root 비밀번호를 설정합니다.


6. **완료**: `MySQL Workbench`를 실행하여 접속 테스트를 진행합니다.

## 공공부문 DB 주요 현황

행정안전부의 '2025년도 범정부EA기반 공공부문 정보자원 현황 통계보고서'에 따르면, 공공 시장은 여전히 외산 DB의 비중이 높지만 국산 및 오픈소스 DB의 점유율이 빠르게 상승하고 있습니다.

### 1. 주요 시장 점유율 (2024년 말 기준)

공공기관에 도입된 주요 DBMS 공급업체 상위 5곳은 다음과 같습니다.

| 순위 | 공급업체 (제품명) | 점유율 (%) | 비고 |
| --- | --- | --- | --- |
| **1위** | **Oracle** | 62.51% | 압도적 1위이나 비중 점진적 하락 중 |
| **2위** | **Microsoft (SQL Server)** | 14.36% | 외산 DB 중 2위 |
| **3위** | **큐브리드 (CUBRID)** | 10.58% | **국산 1위**, 오픈소스 기반 |
| **4위** | **티맥스티베로 (Tibero)** | 9.07% | **국산**, Oracle 호환성 강점 |

## 핵심 시스템별 DB 사용 사례

### 1. 국산 DB (Tibero, CUBRID)

정부 주도의 '국산 SW 권장 정책'과 클라우드 전환 가속화에 따라 사용량이 급증하고 있습니다.

* **티베로 (Tibero)**: 현대자동차, 현대중공업 등 대기업뿐만 아니라 **기획재정부 차세대 예산회계시스템(디지털예산회계시스템)** 등 대규모 행정 시스템에 적용되었습니다. Oracle과 높은 호환성을 가져 기존 시스템 전환 시 선호됩니다.
* **큐브리드 (CUBRID)**: 오픈소스 기반으로 **정부통합전산센터(NIRS)**의 표준 DB 중 하나로 지정되어 있습니다. 국방부, 행정안전부 등 수많은 공공기관의 웹 서비스 및 업무 시스템에서 사용됩니다.

### 2. 외산 DB (Oracle, MS SQL)

* **Oracle**: 안정성과 고성능이 필수적인 **주민등록 시스템, 대법원 등기 시스템** 등 국가 핵심 기반 데이터베이스에서 주로 사용됩니다.
* **MS SQL Server**: 윈도우 기반 환경의 소규모 기관이나 특정 교육/행정 서비스에서 주로 활용됩니다.

### 3. 오픈소스 DB (MySQL, MariaDB, PostgreSQL)

* **전자정부 표준프레임워크**: 한국 공공 프로젝트의 표준인 이 프레임워크는 기본적으로 **MySQL, PostgreSQL, MariaDB**와의 연동을 공식 지원합니다.
* 최근 공공 클라우드(Cloud) 전환 사업이 활발해지면서 라이선스 비용이 없는 오픈소스 DB 도입이 활발합니다.

## 클라우드 및 비용(과금) 주의 사항

공공기관이 **AWS(Amazon Web Services)** 또는 **네이버 클라우드(NCP)**와 같은 민간 클라우드를 사용할 경우 다음과 같은 비용이 발생합니다.

* **RDS (Managed DB) 서비스**:
* **MySQL, PostgreSQL, MariaDB** 등을 선택하면 시간당 인스턴스 비용이 발생합니다.
* **Oracle/SQL Server**를 클라우드에서 쓸 경우 라이선스 비용이 포함된 고가의 요금이 부과되거나, 직접 라이선스를 가져와야(BYOL) 합니다.


* **데이터 전송료**: DB 데이터를 외부로 추출하거나 다른 지역(Region)으로 보낼 때 데이터 전송 비용이 발생합니다.
