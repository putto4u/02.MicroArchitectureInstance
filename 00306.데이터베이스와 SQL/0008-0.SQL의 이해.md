# SQL 데이터베이스 입문

## 데이터베이스와 SQL의 이해

데이터베이스(Database)는 체계적으로 관리되는 데이터의 집합입니다. 우리가 일상적으로 사용하는 앱이나 웹 서비스의 회원 정보, 게시글, 주문 기록 등은 모두 데이터베이스에 저장됩니다.

**SQL(Structured Query Language)**은 이러한 관계형 데이터베이스 관리 시스템(RDBMS)과 소통하기 위한 표준 언어입니다.

---

## 데이터베이스 구조의 핵심 요소

1. **Table(테이블)**: 데이터를 저장하는 기본 단위로, 행과 열로 구성된 표 형태입니다.
2. **Column(컬럼/열)**: 데이터의 속성을 정의합니다. (예: 이름, 나이, 이메일)
3. **Row(로우/행)**: 실제 데이터 한 건을 의미합니다. (예: 홍길동, 25세, hong@example.com)
4. **Primary Key(기본키)**: 각 행을 유일하게 식별할 수 있는 고유값입니다.

---

## 필수 SQL 구문 (DML: Data Manipulation Language)

데이터를 조회하고 조작할 때 가장 많이 사용되는 4가지 핵심 구문입니다.

### 1. SELECT (데이터 조회)

테이블에서 원하는 데이터를 선택하여 가져옵니다.

```sql
-- 모든 컬럼 조회
SELECT * FROM users;

-- 특정 컬럼(이름, 이메일)만 조회
SELECT name, email FROM users;

```

### 2. WHERE (조건 필터링)

특정 조건에 맞는 데이터만 추출할 때 사용합니다.

```sql
-- 나이가 20세 이상인 사용자 조회
SELECT * FROM users 
WHERE age >= 20;

-- 이름이 '김철수'인 사용자 조회
SELECT * FROM users 
WHERE name = '김철수';

```

### 3. INSERT (데이터 삽입)

테이블에 새로운 행을 추가합니다.

```sql
INSERT INTO users (name, age, email) 
VALUES ('이영희', 30, 'young@example.com');

```

### 4. UPDATE (데이터 수정)

기존에 저장된 데이터를 변경합니다. **주의: WHERE 절을 생략하면 모든 데이터가 수정됩니다.**

```sql
UPDATE users 
SET age = 31 
WHERE name = '이영희';

```

### 5. DELETE (데이터 삭제)

데이터를 삭제합니다. **주의: WHERE 절이 없으면 테이블의 모든 데이터가 삭제됩니다.**

```sql
DELETE FROM users 
WHERE id = 5;

```

## 데이터 정렬과 제한

데이터를 보기 좋게 정렬하거나 필요한 만큼만 끊어서 가져오는 방법입니다.

* **ORDER BY**: 데이터를 정렬합니다. (`ASC`: 오름차순, `DESC`: 내림차순)
* **LIMIT**: 출력할 행의 개수를 제한합니다.

```sql
-- 나이가 많은 순서대로 5명만 조회
SELECT * FROM users 
ORDER BY age DESC 
LIMIT 5;

```

---

## 집계 함수 (Aggregate Functions)

데이터의 통계를 구할 때 사용합니다.

| 함수 | 설명 |
| --- | --- |
| **COUNT()** | 행의 개수를 세어줌 |
| **SUM()** | 수치 컬럼의 합계를 구함 |
| **AVG()** | 수치 컬럼의 평균을 구함 |
| **MAX() / MIN()** | 최대값 / 최소값을 구함 |

```sql
-- 전체 사용자 수 계산
SELECT COUNT(*) FROM users;

-- 사용자들의 평균 나이 계산
SELECT AVG(age) FROM users;

```

## 클라우드 환경의 데이터베이스 (AWS)

실무에서는 로컬 PC보다 클라우드 환경에서 데이터베이스를 운영하는 경우가 많습니다.

* **AWS RDS (Relational Database Service)**: MySQL, PostgreSQL, Oracle 등을 클라우드에서 쉽게 운영할 수 있게 해주는 관리형 서비스입니다.
* **과금 주의**: 프리티어(Free Tier) 범위를 초과하거나, 인스턴스를 종료하지 않고 계속 켜두면 **유료 과금**이 발생합니다.


* **AWS Aurora**: AWS에서 제공하는 고성능 관계형 데이터베이스입니다.
* **과금 주의**: 성능이 강력한 만큼 설정에 따라 **높은 비용**이 발생할 수 있으므로 사용 후 반드시 삭제가 필요합니다.
