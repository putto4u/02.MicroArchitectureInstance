## 3대 클라우드 주요 서비스 및 CLI 명령어 비교

클라우드 3사(AWS, Azure, GCP)는 동일한 기술 개념을 각기 다른 브랜드 명칭(Service Name)으로 정의합니다. 또한 웹 콘솔이 아닌 터미널 환경에서 자원을 제어하는 CLI(Command Line Interface) 명령어 구조에도 차이가 있습니다. 실무에서 혼동하기 쉬운 용어와 명령어를 비교 정리합니다.

### 1. 주요 서비스 명칭 및 용어 비교

각 클라우드의 핵심 서비스 매핑 테이블입니다. 기능적으로 거의 동일한 역할을 수행하는 서비스들을 같은 행에 배치하였습니다.

| 카테고리 | AWS (Amazon) | Microsoft Azure | Google Cloud (GCP) |
| --- | --- | --- | --- |
| **가상 서버** (Compute) | **EC2** (Elastic Compute Cloud) | **Virtual Machines** (VM) | **Compute Engine** (GCE) |
| **객체 스토리지** (File) | **S3** (Simple Storage Service) | **Blob Storage** | **Cloud Storage** (GCS) |
| **블록 스토리지** (Disk) | **EBS** (Elastic Block Store) | **Azure Disk** (Managed Disk) | **Persistent Disk** |
| **네트워크** (Virtual Net) | **VPC** (Virtual Private Cloud) | **VNet** (Virtual Network) | **VPC** (전역 네트워크) |
| **데이터베이스** (RDBMS) | **RDS** (Relational Database) | **Azure SQL Database** | **Cloud SQL** |
| **데이터베이스** (NoSQL) | **DynamoDB** | **Cosmos DB** | **Firestore / Bigtable** |
| **서버리스** (FaaS) | **AWS Lambda** | **Azure Functions** | **Cloud Functions** |
| **컨테이너** (K8s) | **EKS** (Elastic Kubernetes) | **AKS** (Azure Kubernetes) | **GKE** (Google Kubernetes) |
| **로드 밸런서** | **ELB** (Elastic Load Balancing) | **Azure Load Balancer** | **Cloud Load Balancing** |
| **인프라 자동화** (IaC) | **CloudFormation** | **ARM Templates** / Bicep | **Deployment Manager** |

---

### 2. CLI(Command Line Interface) 도구 및 명령어 구조 비교

클라우드 자동화를 위해 필수적인 CLI 도구의 설치 명칭과 기본 문법 구조(Syntax)의 차이입니다.

#### 2.1 CLI 도구 개요

| 구분 | AWS CLI | Azure CLI | Google Cloud SDK |
| --- | --- | --- | --- |
| **실행 명령어** | `aws` | `az` | `gcloud` |
| **문법 구조** | `aws <서비스> <명령>` | `az <그룹> <하위그룹> <명령>` | `gcloud <그룹> <하위그룹> <명령>` |
| **설치 패키지** | AWS CLI v2 | Azure CLI | Google Cloud SDK |
| **출력 형식** | JSON, Text, Table, YAML | JSON, Table, TSV, YAML | JSON, Text, Table, YAML |

#### 2.2 주요 작업별 명령어 예시

실무에서 가장 빈번하게 사용되는 '로그인', '서버 조회', '스토리지 조회' 명령어 비교입니다.

| 작업 (Action) | AWS (`aws`) | Azure (`az`) | GCP (`gcloud`) |
| --- | --- | --- | --- |
| **로그인 및 인증** | `aws configure` | `az login` | `gcloud auth login` |
| **가상 서버 목록 조회** | `aws ec2 describe-instances` | `az vm list` | `gcloud compute instances list` |
| **스토리지 버킷 조회** | `aws s3 ls` | `az storage account list` | `gcloud storage ls` |
| **리소스 생성 도움말** | `aws ec2 run-instances help` | `az vm create --help` | `gcloud compute instances create --help` |

---

### 3. CLI 사용 시 주의사항 (과금 및 구성)

CLI는 웹 콘솔보다 실행 속도가 빠르기 때문에, 실수로 유료 자원을 대량 생성하거나 삭제하지 못하고 방치할 위험이 높습니다.

1. **자격 증명 관리:**
* **AWS:** `aws configure`로 입력한 Access Key/Secret Key가 `~/.aws/credentials` 파일에 평문으로 저장됩니다. 이 파일이 깃허브 등에 업로드되지 않도록 주의해야 합니다.
* **GCP:** `gcloud auth login`은 웹 브라우저 기반 인증(OAuth)을 사용하므로 상대적으로 안전하지만, 서비스 계정 키(JSON) 파일 관리에 유의해야 합니다.


2. **프로젝트/리전 설정:**
* 명령어 실행 시 대상 리전(Region)이나 프로젝트(Project)를 명시하지 않으면 기본값(Default) 설정에 따라 엉뚱한 곳에 자원이 생성될 수 있습니다.
* *예시:* AWS에서 `ap-northeast-2`(서울)가 아닌 `us-east-1`에 고성능 인스턴스를 생성하여 과금 발생.


3. **스크립트 실행 주의:**
* 반복문(Loop) 스크립트를 잘못 작성하여 유료 인스턴스를 수백 개 생성하는 사고(Bill Shock)가 발생할 수 있습니다. 스크립트 실행 전에는 반드시 `--dry-run` 옵션(지원하는 경우)을 사용하여 테스트해야 합니다.
