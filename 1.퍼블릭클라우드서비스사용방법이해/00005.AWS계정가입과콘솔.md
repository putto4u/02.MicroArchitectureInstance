## AWS 계정 생성 및 필수 초기 보안/비용 설정

AWS 클라우드를 시작하기 위한 계정 생성부터, 보안을 위한 계정 분리(Root vs IAM), 그리고 가장 중요한 요금 폭탄 방지 설정까지의 필수 절차를 다룹니다.

---

### 1. AWS 회원 가입 (Account Creation)

AWS 가입은 해외 결제가 가능한 신용/체크카드와 본인 인증 수단(휴대폰)이 필요합니다.

1. **접속:** [AWS 공식 홈페이지](https://www.google.com/search?q=https://aws.amazon.com/ko/) 우측 상단의 **[AWS 계정 생성]** 클릭.
2. **루트 사용자 이메일 주소:** 로그인 아이디로 사용할 이메일 주소를 입력합니다.
3. **암호 설정:** 강력한 규칙(대/소문자, 특수문자 포함)의 암호를 설정합니다.
4. **연락처 정보:** 개인(Personal) 또는 비즈니스(Business)를 선택하고 영문 주소를 입력합니다. (네이버 영문 주소 변환 활용 권장)
5. **결제 정보(카드) 입력:** 해외 결제 승인 확인을 위해 1달러(또는 100원)가 가결제되었다가 며칠 뒤 취소됩니다.
6. **자격 증명 확인:** 휴대폰 ARS 또는 SMS 문자로 본인 인증을 수행합니다.
7. **지원 플랜 선택:** 학습 목적이라면 무료인 **'기본 지원(Basic Support) - Free'**를 선택합니다.

> **※ 프리 티어(Free Tier):** 신규 가입 시 12개월간 EC2(t2.micro/t3.micro), RDS 등 특정 리소스를 무료 한도 내에서 제공합니다. 한도를 초과하면 즉시 과금되므로 주의가 필요합니다.

---

### 2. Root 계정과 IAM 사용자(User)의 차이

AWS에는 두 가지 유형의 사용자가 존재하며, 이 둘의 권한과 용도를 명확히 구분해야 합니다.

| 구분 | 루트 사용자 (Root User) | IAM 사용자 (IAM User) |
| --- | --- | --- |
| **생성 시점** | 회원 가입 시 자동 생성 | Root 또는 관리자가 별도 생성 |
| **로그인 ID** | **이메일 주소** | **계정 ID (12자리 숫자)** + 사용자명 |
| **권한 범위** | **무제한 (모든 리소스 제어/삭제/결제)** | 부여된 권한(Policy) 내에서만 활동 |
| **보안 위험** | 해킹 시 계정 전체 탈취 및 막대한 피해 | 해킹 시 해당 사용자의 권한만 위험 |
| **사용 원칙** | **평소 사용 금지** (금고 열쇠처럼 보관) | **일상적인 작업 및 개발에 사용** |
| **주요 용도** | 계정 해지, 결제 정보 변경, IAM 관리자 생성 | EC2 생성, S3 관리, 개발, 운영 등 |

---

### 3. 필수 보안 설정: MFA 설정 및 관리자(Admin) 생성

가입 직후 Root 계정으로 로그인하여 가장 먼저 해야 할 일은 **Root 계정 잠금(MFA)**과 **작업용 관리자 계정 생성**입니다.

#### 3.1 Root 계정 MFA(다중 인증) 설정

MFA(Multi-Factor Authentication)는 로그인 시 비밀번호 외에 OTP(Google Authenticator 등)를 한 번 더 입력하게 하는 이중 보안 장치입니다.

1. Root 계정(이메일)으로 로그인.
2. 우측 상단 계정명 클릭 > **[보안 자격 증명(Security Credentials)]** 선택.
3. **MFA(멀티 팩터 인증)** 섹션 > **[MFA 할당]** 클릭.
4. **[인증 관리자 앱]** 선택 후 스마트폰에 'Google Authenticator' 설치 및 QR 코드 스캔.
5. 생성된 연속된 두 개의 인증 코드(6자리)를 입력하여 활성화.

#### 3.2 IAM 관리자 사용자 만들기 (실습용 계정)

앞으로는 Root가 아닌 이 계정으로 로그인하여 실습합니다.

1. 콘솔 상단 검색창에 `IAM` 검색 후 선택.
2. 좌측 메뉴 **[사용자(Users)]** > **[사용자 생성]** 클릭.
3. **사용자 세부 정보:**
* 사용자 이름: `admin` (또는 본인 닉네임)
* **AWS Management Console에 대한 사용자 액세스 권한 제공** 체크.
* **IAM 사용자를 생성하고 싶음** 선택.
* 콘솔 비밀번호 설정 (자동 생성 또는 사용자 지정).


4. **권한 설정:**
* **[직접 정책 연결]** 선택.
* 권한 정책 목록에서 `AdministratorAccess` 검색 후 체크. (모든 서비스 접근 가능 권한)


5. **검토 및 생성:** 설정을 확인하고 생성 완료.
6. **중요:** `.csv` 파일을 다운로드하거나 **콘솔 로그인 URL**(https://[계정ID][.signin.aws.amazon.com/console)을](https://www.google.com/search?q=https://.signin.aws.amazon.com/console)을) 복사해 둡니다.

### 4. AWS 관리 콘솔(Management Console) 주요 메뉴 및 위치

`admin` 계정으로 로그인했을 때 보이는 메인 화면의 구조입니다.

#### 4.1 상단 네비게이션 바 (Global Menu)

* **서비스 검색 (Alt+S):** 원하는 서비스(EC2, S3, RDS 등)를 검색하여 이동합니다.
* **CloudShell (>_):** 별도 설치 없이 브라우저에서 즉시 AWS CLI를 사용할 수 있는 리눅스 터미널을 엽니다.
* **리전(Region) 선택기:** **[매우 중요]** 현재 리소스를 생성할 물리적 위치를 선택합니다. 한국에서는 반드시 **`아시아 태평양(서울) ap-northeast-2`**를 선택해야 속도가 빠르고 관리가 용이합니다.
* *주의:* 실수로 '버지니아 북부' 등에 리소스를 만들고 서울에서 찾으면 보이지 않습니다.


* **사용자 메뉴:** 계정 ID(12자리), 보안 자격 증명, 로그아웃 기능이 있습니다.

#### 4.2 메인 대시보드

* **최근 방문:** 최근에 사용한 서비스 목록이 표시됩니다.
* **AWS Health:** AWS 전체 인프라의 장애 여부나 계정 관련 알림이 표시됩니다.
* **비용 및 사용량:** 현재까지 발생한 요금의 개략적인 정보를 보여줍니다.

### 5. 요금 폭탄 방지: AWS Budgets 설정

실수로 리소스를 켜두거나 해킹으로 인해 과도한 요금이 청구되는 것을 막기 위한 안전장치입니다. 설정한 금액을 넘을 것 같으면 이메일로 경고를 보냅니다.

#### 설정 방법

1. 콘솔 검색창에 `Budgets` (AWS Budgets) 검색 및 이동.
2. **[예산 생성(Create budget)]** 클릭.
3. **[예산 설정 사용자 지정(Advanced)]** > **[비용 예산]** 선택.
4. **세부 정보:**
* 기간: **월별(Monthly)**
* 예산 유형: **고정(Fixed)**
* 예산 금액: `$10.00` (본인이 허용 가능한 최소 금액, 예: 10달러)


5. **알림 임계값 구성 (Alerts):**
* 임계값: **80%** (예산의 80%인 8달러 도달 시 알림)
* 이메일 수신자: 본인의 이메일 주소 입력.


6. **[예산 생성]** 완료.

> **Tip:** AWS Budgets는 단순히 '알림'만 줍니다. 실제로 서비스가 자동으로 꺼지지는 않으므로, 알림 메일을 받으면 즉시 콘솔에 접속하여 불필요한 리소스를 종료해야 합니다.
