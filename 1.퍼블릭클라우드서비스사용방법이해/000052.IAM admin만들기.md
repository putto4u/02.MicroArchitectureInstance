## 루트(Root) 계정 가입 후 IAM 초기 설정 가이드

AWS 루트 계정은 해당 AWS 계정의 모든 리소스에 대한 **무제한 권한**을 가집니다. 따라서 해킹 시 막대한 금전적 피해(비트코인 채굴 등)를 입을 수 있으므로, 루트 계정은 '금고'처럼 잠가두고 평소에는 권한이 제한된 'IAM 사용자'를 생성하여 사용하는 것이 원칙입니다.

### 1. 루트 계정과 IAM 사용자 비교

| 구분 | 루트 사용자 (Root User) | IAM 관리자 (IAM Admin User) |
| --- | --- | --- |
| **로그인 ID** | 이메일 주소 | 사용자 지정 ID (예: admin, gildong) |
| **권한 범위** | **무제한 (제어 불가능)** | 부여된 권한만큼만 행사 (제어 가능) |
| **주요 용도** | 계정 해지, 결제 정보 변경, Support Plan 변경 | 리소스 생성, 삭제, 일반적인 운영 업무 |
| **보안 권장** | **MFA 설정 후 봉인 (사용 금지)** | MFA 설정 후 일상적으로 사용 |

---

### 2. 필수 초기 설정 3단계 프로세스

가입 직후 아래 3가지 작업을 반드시 수행해야 보안 사고를 예방할 수 있습니다.

#### 1단계: 루트 계정 MFA(다중 인증) 설정

비밀번호가 유출되더라도 OTP 없이는 로그인할 수 없도록 이중 잠금장치를 겁니다.

1. 콘솔 우측 상단 계정 이름 클릭 -> **보안 자격 증명(Security Credentials)** 선택.
2. **MFA(멀티 팩터 인증)** 섹션에서 'MFA 할당' 클릭.
3. 스마트폰에 `Google Authenticator` 앱 설치 후 QR 코드 스캔 및 인증 번호 입력.

#### 2단계: IAM 관리자(Admin) 사용자 생성

루트 계정을 대신해 평소에 사용할 '관리자급' 사용자를 만듭니다.

1. 서비스 검색창에 `IAM` 입력 후 이동.
2. **사용자(Users)** -> **사용자 생성(Create user)** 클릭.
3. 사용자 이름 입력 (예: `admin-user`).
4. **AWS Management Console 액세스 권한 제공** 체크 -> 'IAM 사용자를 생성하고 싶습니다' 선택.
5. **권한 옵션:** '직접 정책 연결' 선택 후 **`AdministratorAccess`** 정책 검색하여 체크.
6. 생성 완료 후 나오는 **콘솔 로그인 URL**, **사용자 이름**, **비밀번호**를 별도로 저장 (.csv 다운로드 권장).

#### 3단계: 계정 별칭(Alias) 생성 (선택 사항)

IAM 사용자로 로그인할 때, 복잡한 12자리 계정 번호 대신 알아보기 쉬운 별칭을 URL에 사용합니다.

1. IAM 대시보드 우측의 **AWS 계정(AWS Account)** 섹션 확인.
2. **계정 별칭(Account Alias)** -> '생성' 클릭.
3. 원하는 영문 이름 입력 (예: `my-company-cloud`).
4. 로그인 URL이 `https://my-company-cloud.signin.aws.amazon.com/console`로 변경됨.

---

### 3. 비용 정보 (Cost)

* **IAM 서비스 자체:** **무료 ($0)**
* 사용자(User), 그룹(Group), 역할(Role), 정책(Policy)을 생성하고 관리하는 기능은 AWS 사용료에 포함되지 않는 무료 기능입니다.
* 보안 강화를 위해 사용자를 아무리 많이 만들어도 추가 비용은 없습니다.



---

### 4. 이후 로그인 방법

설정이 완료되면 더 이상 이메일(루트)로 로그인하지 않습니다.

1. 브라우저 주소창에 **계정 별칭이 포함된 URL** 입력.
2. **계정 ID(별칭)**, **IAM 사용자 이름**, **비밀번호** 입력.
3. 로그인 후 우측 상단을 확인했을 때 `admin-user @ my-company-cloud` 형식으로 표시되면 성공입니다.
