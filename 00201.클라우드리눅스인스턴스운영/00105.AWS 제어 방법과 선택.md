# AWS 제어 인터페이스 종류와 선택 가이드

## 1. 개요

결론부터 말하자면 **AWS CLI 설치가 필수는 아니다.**

AWS의 모든 리소스 제어는 근본적으로 **AWS API**를 호출하는 방식으로 이루어진다. 사용자는 이 API를 호출하기 위해 CLI 외에도 웹 콘솔(GUI), SDK, IaC 도구 등 다양한 인터페이스를 선택할 수 있다. 작업의 목적, 숙련도, 자동화 필요 여부에 따라 적절한 도구를 선택하면 된다.

---

## 2. AWS 리소스 제어 인터페이스 유형

AWS 리소스를 생성하고 관리하는 방법은 크게 4가지로 분류된다.

### 2.1 AWS Management Console (웹 GUI)

* **방식:** 웹 브라우저에서 마우스를 사용하여 버튼을 클릭하고 폼을 입력하는 방식.
* **장점:**
* 직관적인 UI/UX로 시각적 확인이 용이하다.
* 초보자가 학습하거나 구조를 파악하기에 가장 좋다.
* 별도의 설치 과정이 전혀 필요 없다.


* **단점:**
* 반복 작업 시 실수(Human Error)가 발생하기 쉽다.
* 수백 대의 서버를 동시에 구축하는 등의 대규모 작업이 불가능에 가깝다.
* 작업 이력을 코드로 남길 수 없어 히스토리 관리가 어렵다.



### 2.2 AWS CloudShell (웹 기반 CLI)

* **방식:** AWS 웹 콘솔 하단에서 실행되는 브라우저 기반의 터미널.
* **특징:**
* `aws`, `python`, `node.js` 등 주요 도구가 사전 설치되어 있다.
* 로컬 PC에 설치하지 않아도 CLI 명령어를 바로 실행할 수 있다.


* **제약:**
* 세션 유지 시간이 존재하며, 파일 저장이 영구적이지 않다(최대 1GB, 120일 미사용 시 삭제).



### 2.3 AWS CLI (Local Command Line Interface)

* **방식:** 사용자 PC의 터미널에서 텍스트 명령어로 AWS를 제어.
* **장점:**
* 스크립트 작성을 통한 **자동화**가 가능하다.
* GUI보다 속도가 빠르고 명확하다.
* 다른 프로그램(Git, Jenkins 등)과 연계하여 파이프라인 구축이 가능하다.



### 2.4 IaC (Infrastructure as Code) 및 SDK

* **방식:** Terraform, Ansible, CloudFormation 같은 코드형 인프라 도구 또는 Boto3(Python SDK) 같은 라이브러리 사용.
* **특징:**
* 현대적인 클라우드 운영의 **표준**이다.
* CLI 명령어조차 추상화하여, 인프라의 상태를 코드 파일로 정의하고 관리한다.
* 내부적으로는 AWS API를 호출하지만, 사용자가 직접 `aws ec2 run-instances...` 처럼 명령어를 칠 필요가 없다.

## 3. 도구별 비교 요약

| 구분 | **웹 콘솔 (GUI)** | **로컬 AWS CLI** | **AWS CloudShell** | **IaC (Terraform 등)** |
| --- | --- | --- | --- | --- |
| **설치 필요 여부** | 불필요 | **필요** | 불필요 | **필요** |
| **진입 장벽** | 매우 낮음 | 중간 | 중간 | 높음 |
| **자동화 적합성** | 부적합 | **높음** | 중간 | **최상** |
| **재현성** | 낮음 (매번 클릭 필요) | 높음 (스크립트) | 높음 (스크립트) | 최상 (코드 기반) |
| **주 사용 목적** | 학습, 모니터링, 단발성 작업 | 스크립팅, 자동화, 운영 | 긴급 수정, 간단한 조회 | 대규모 인프라 구축 및 관리 |

> **Cost Alert (과금 주의):**
> 어떤 도구를 사용하든 생성되는 **리소스(EC2, RDS 등)의 가격은 동일하게 부과**된다. "무료 툴인 콘솔을 썼으니 무료"가 아니며, 명령을 내리는 수단만 다를 뿐 결과물에 대한 비용은 사용자 몫이다.

---

## 4. 결론: 실무에서의 선택

1. **초기 학습 및 구조 파악:** 웹 콘솔(GUI) 사용이 권장된다. 시각적으로 리소스를 확인하며 흐름을 익히는 데 유리하다.
2. **반복 작업 및 운영:** AWS CLI 설치가 **사실상 필수**가 된다. 매일 서버를 켜고 끄거나, 로그를 조회하는 단순 반복 업무를 GUI로 하는 것은 비효율적이다.
3. **대규모 인프라 구축:** 최종적으로는 CLI를 넘어 **Terraform이나 CloudFormation 같은 IaC 도구**를 사용하게 된다. 하지만 이 도구들도 내부 작동 원리는 CLI와 유사하므로 CLI에 대한 이해가 바탕이 되어야 한다.

따라서, **"작업 자체는 가능하지만, 엔지니어로서 커리어를 쌓으려면 CLI 설치 및 사용법 숙지는 필수"**이다.
