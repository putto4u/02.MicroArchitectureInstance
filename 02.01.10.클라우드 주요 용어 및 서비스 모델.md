## 클라우드 주요 용어 및 서비스 모델 요약 (업데이트)

| 구분 | 용어 | 설명 | 비고 (과금 및 주의사항) |
| --- | --- | --- | --- |
| **인프라 구성** | **Region (리전)** | 클라우드 서비스가 물리적으로 제공되는 지리적 거점 | 서비스 속도를 위해 사용자 근접 리전 선택 권장 |
|  | **Availability Zone (AZ)** | 리전 내에서 전력·네트워크가 독립된 데이터 센터 군 | 고가용성 및 재해 복구(DR) 설계의 핵심 |
| **서비스 모델** | **IaaS** | 가상 서버, 스토리지, 네트워크 등 기초 인프라 제공 | **[유료]** 인스턴스 사양 및 가동 시간당 과금 |
|  | **PaaS** | 앱 개발에 필요한 런타임, DB 등 플랫폼 환경 제공 | 인프라 관리 부담 없이 개발에 집중 가능 |
|  | **SaaS** | 완성된 소프트웨어를 네트워크를 통해 즉시 사용 | 월간/연간 구독료 형태의 비용 발생 |
| **컴퓨팅** | [**Instance**(인스턴스) ](https://github.com/putto4u/AWS/blob/main/0011.%EC%9D%B8%EC%8A%A4%ED%84%B4%EC%8A%A4%EC%99%80%20%EC%83%9D%EB%AA%85%EC%A3%BC%EA%B8%B0.md)| 클라우드에서 실행되는 가상 서버 단위 | **[유료]** 실행 중인 시간에 비례하여 비용 발생 |
| **네트워크** | **VPC** | 클라우드 내 논리적으로 격리된 사용자 전용 가상 네트워크 | 보안을 위한 서브넷 및 라우팅 설정 가능 |
|  | **Load Balancer** | 여러 서버로 트래픽을 분산하여 안정성을 높이는 장치 | **[유료]** 시간당 고정비 및 처리 데이터량 과금 |
| **운영 및 저장** | **Auto Scaling** | 부하량에 따라 서버 자원을 자동으로 증설/감소 | 급격한 트래픽 변화에 유연하게 대응 |
|  | **Object Storage** | 파일을 객체 단위로 저장하며 URL로 접근하는 저장소 | 저장 용량 및 데이터 전송량(Outbound) 과금 |
|  | **Serverless** | 서버 관리 없이 코드 실행 시에만 자원을 할당받는 방식 | 코드 실행 횟수 및 실행 시간만큼만 비용 지불 |
